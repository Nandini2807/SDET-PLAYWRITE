1. setTimeout:
        setTimeout() is used to execute a function once after a specific delay.

- Example:
        setTimeout(() => {
        console.log("Runs after 2 sec");
        }, 2000);

- Key Points:
Delay is in milliseconds
Executes only once
Non-blocking (runs in background)

* ** * * * * * * ** * * * ** * * ** * * * * * * ** ** * ** * ** * * ** ** * * * 

2. setInterval:
        setInterval() runs a function repeatedly after a fixed interval.

- Example:
        let id = setInterval(() => {
        console.log("Runs every 1 sec");
        }, 1000);

- stop interval:
        clearInterval(id);


clearInterval() stops execution
If used immediately, interval may not run
 
- Better Usage:
        setTimeout(() => {
        clearInterval(id);
        }, 5000);

* * * * * *  * * * * * ** * * * * * ** ** * * ** * * ** * * * ** * * * * * * * ** 

3. Callback Function:
      A callback is a function passed as an argument to another function and executed later.

- Example:
     function getData(callback) {
     setTimeout(() => {
     console.log("Data fetched");
     callback();
     }, 1000);
     }

     getData(() => console.log("Next step"));

 Output:
     Data fetched
     Next step

- Limitation:
     Can lead to callback hell (nested callbacks)

* * ** * * * ** * ** * * ** * * * * ** * ** * * * * * * ** * * ** * * * * * * * * *

4. Promises:
     A Promise represents the result of an asynchronous operation.

- States:
     Pending
     Resolved
     Rejected

- Example:
     let myPromise = new Promise((resolve, reject) => {
     let success = true;

     if (success) resolve("Success");
     else reject("Error");
     });

- Handling:
myPromise
    .then(result => console.log(result))
    .catch(error => console.log(error));

* * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * * *  * * * * * * * * *   

- fetching APIS:
     To fetch an API in JavaScript, you are essentially asking a remote server for data. Because the server might be across the world, this process is asynchronous—your code starts the request and then "waits" for the data to come back without stopping the rest of your program.

example:

fetch("https://fakestoreapi.com/products/1")
    .then(res => res.json())        // Convert response
    .then(data => console.log(data)) // Use data
    .catch(err => console.log(err)); // Handle errors

- The Response Object (res):

A. The first thing fetch returns is not your data; it's a Response object.

It tells you if the request was successful (res.ok).

It tells you the status code (200 for success, 404 for not found).
B. JSON Conversion
Computers send data over the internet as a stream of "chunks." The .json() method is necessary because it tells JavaScript to wait for all the chunks to arrive and then turn them into an object you can actually use.
C. Error Handling
Always use .catch() or try/catch. Without it, if the API is down or the user loses Wi-Fi, your entire application might crash.

📘 Day 05 Summary: Async/await Function and OPPs Concept JavaScript

1. Async/Await:
     Async/await is used to handle asynchronous operations in a simple and readable way. 
     The async keyword defines a function that returns a Promise, 
     while await pauses execution until the Promise is resolved.
      This makes code easier to understand compared to .then().

- Example:
    async function getData() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.log("Error:", error);
    }
}

getData();

- Key Learnings:
     Async/Await simplifies asynchronous programming and improves code readability in JavaScript.
     Helps avoid complex .then() chaining
     Makes asynchronous code simple and easy to read like synchronous code
     await pauses execution until the Promise resolves
     async functions always return a Promise

3.OOPs concepts

1. Encapsulation:
       Encapsulation means hiding data and restricting direct access using private variables.
       Data is accessed through methods only.

Example:
       class Bank {
    #balance = 0; // private

    deposit(amount) {
        this.#balance += amount;
    }

    getBalance() {
        return this.#balance;
    }
}

Point:
     Protects data from wrong use
     Uses private variables (#)

2. Abstraction:
       Abstraction means hiding internal details and showing only necessary features.

Example:
       class CoffeeMachine {
    #boilWater() {
        console.log("Boiling water");
    }

    makeCoffee() {
        this.#boilWater();
        console.log("Coffee ready");
    }
}

Point:
      User only calls makeCoffee()
      Internal steps are hidden

3. Inheritance:
       Inheritance means one class can use properties/methods of another class.

Example:
       class Animal {
    speak() {
        console.log("Animal makes sound");
    }
}

class Dog extends Animal {
    bark() {
        console.log("Dog barks");
    }
}

const d = new Dog();
d.speak(); // inherited
d.bark();

Point:
      Reuse code
      Avoid duplication      


4. Polymorphism:
       Polymorphism means same method behaves differently in different classes.

Example:
       class Animal {
    speak() {
        console.log("Animal sound");
    }
}

class Dog extends Animal {
    speak() {
        console.log("Bark");
    }
}

class Cat extends Animal {
    speak() {
        console.log("Meow");
    }
}

new Dog().speak(); // Bark
new Cat().speak(); // Meow

Point:
      Same method name → different behavior

Question 1: Flight Booking Data Cleaner

#Problem Statement:
------------------
       Raw flight data is provided in string format:
       "CITY_A-CITY_B:PRICE"


The task is to:
       * Convert strings into objects
       * Validate prices
       * Filter flights within a price range
       * Sort flights
       * Return JSON string


# Code:
-------

       const rawFlights = [
  "London-Paris:150",
  "New York-Tokyo:invalid",
  "Dubai-Mumbai:450",
  "Berlin-Rome:95"
];

function cleanFlightData(flights) {

  const result = flights

    .map(flight => {

      if (!flight || typeof flight !== "string") {
        return {
          from: "",
          to: "",
          price: 0
        };
      }

      const [route, rawPrice] = flight.split(":");
      const [from, to] = route.split("-");

      let price = Number(rawPrice);

      if (isNaN(price)) {
        price = 0;
      }

      return {
        from: from || "",
        to: to || "",
        price
      };
    })

    .filter(flight => flight.price >= 100 && flight.price <= 500)

    .sort((a, b) => a.price - b.price);

  return JSON.stringify(result, null, 2);
}
console.log(cleanFlightData(rawFlights));

#Explanation:
-------------

       map(): Used to convert strings into objects.
       split: Breaks string into smaller parts.

Example:
       "London-Paris:150"


After split:
       from = "London"
       to = "Paris"
       price = "150"

Number(): Converts string into number.

isNaN(): Checks invalid number values.

filter(): Keeps flights with prices between:
          100 & 500

sort(): Sorts flights from cheapest to highest.

JSON.stringify: Converts JavaScript object into JSON string.

#Final Output:
-------------

[
  {
    "from": "London",
    "to": "Paris",
    "price": 150
  },
  {
    "from": "Dubai",
    "to": "Mumbai",
    "price": 450
  }
]

#Question 2: E-Commerce Discount Applicator

#Problem Statement:
------------------

Create a promo system that:

* Identifies eligible products
* Applies 10% discount
* Adds discount status
* Calculates total savings
* Uses callback function
* Uses `setTimeout()`

---

# Code:
-------

const cart = [
  { name: "Smartphone", price: 800, category: "Electronics" },
  { name: "Toaster", price: 50, category: "Home" },
  { name: "Headphones", price: 250, category: "Electronics" },
  { name: "Monitor", price: 150, category: "Electronics" }
];

function isEligible(product) {
  return (
    product &&
    product.category === "Electronics" &&
    product.price > 200
  );
}

function applyPromo(cart, promoCallback) {

  if (!Array.isArray(cart)) {
    console.log("Invalid cart data");
    return;
  }

  let discountedCount = 0;
  let totalSavings = 0;

  const updatedCart = cart.map(product => {

    if (!product) {
      return {
        name: "",
        price: 0,
        category: "",
        isDiscounted: false
      };
    }

    if (promoCallback(product)) {

      const discount = product.price * 0.10;
      discountedCount++;
      totalSavings += discount;

      return {
        ...product,
        price: product.price - discount,
        isDiscounted: true
      };
    }

    return {
      ...product,
      isDiscounted: false
    };
  });

  console.log(
    `Promotion applied! ${discountedCount} items were discounted for a total saving of $${totalSavings}.`
  );

  setTimeout(() => {
    console.log(updatedCart);
  }, 1000);
}

applyPromo(cart, isEligible);

#Explanation:
-------------

Callback Function:
        function isEligible(product)

Checks:
      * Product category is Electronics
      * Price greater than 200

map():Transforms all products.

Spread Operator:
       ...product
       Copies all existing object properties.

Discount Formula:
       product.price * 0.10
       Applies 10% discount.

#Template Literal:
         `${variable}`: Used for dynamic messages.


setTimeout(): Simulates delayed server processing.
              1000ms = 1 second

#Final Output:
--------------

[
  {
    name: "Smartphone",
    price: 720,
    category: "Electronics",
    isDiscounted: true
  }

# Question 3: Movie Stream Analytics

#Problem Statement:
-------------------

Movie data is provided in string format:
"MOVIE_NAME|GENRE|VIEW_COUNT"


The task is to:

* Convert into objects
* Validate views
* Filter Action & Sci-Fi movies
* Sort by views
* Return JSON string

#Code:
------

const rawMovies = [
  "Inception|Sci-Fi|12000",
  "The Lion King|Animation|8000",
  "Mad Max|Action|invalid",
  "The Matrix|Sci-Fi|15000",
  "Gladiator|Action|4500"
];

function analyzeMovies(movies) {

  const result = movies

    .map(movie => {

      if (!movie || typeof movie !== "string") {
        return {
          name: "",
          genre: "",
          views: 0
        };
      }

      const [name, genre, rawViews] = movie.split("|");

      let views = Number(rawViews);

      if (isNaN(views)) {
        views = 0;
      }

      return {
        name: name || "",
        genre: genre || "",
        views
      };
    })

    .filter(movie =>
      (movie.genre === "Action" || movie.genre === "Sci-Fi") &&
      movie.views > 5000
    )

    .sort((a, b) => b.views - a.views);

  return JSON.stringify(result, null, 2);
}

console.log(analyzeMovies(rawMovies));

#Explanation:
-------------

split("|")
Separates:
        * Movie Name
        * Genre
        * Views

Number(): Converts views into numeric value.

isNaN(): Handles invalid data.

Example:
        "invalid" → 0


filter()
Keeps only:
       * Action movies
       * Sci-Fi movies
       * Views greater than 5000

Descending Sort:
b.views - a.views:
       Shows highest viewed movies first.

#Final Output


[

    "views": 15000

#Question 4: Automated Payroll Processor

#Problem Statement:
-------------------

Create a payroll system that:

* Applies tax deduction
* Calculates net salary
* Adds employee status
* Calculates total payout
* Uses callback function
* Uses delayed processing

#Code:
------

const employees = [
  { id: 101, name: "Alice", salary: 6000 },
  { id: 102, name: "Bob", salary: 3500 },
  { id: 103, name: "Charlie", salary: 5200 }
];

function taxLogic(salary) {

  if (salary > 5000) {
    return 0.20;
  }

  return 0.10;
}

function calculatePayroll(employees, taxCallback) {

  if (!Array.isArray(employees)) {
    console.log("Invalid employee data");
    return;
  }

  let totalNetPayout = 0;

  const processedEmployees = employees.map(emp => {

    if (!emp) {
      return {
        id: 0,
        name: "",
        salary: 0,
        netSalary: 0,
        status: "Standard"
      };
    }

    const taxRate = taxCallback(emp.salary);

    const tax = emp.salary * taxRate;

    const netSalary = emp.salary - tax;

    totalNetPayout += netSalary;

    const status =
      netSalary > 4000 ? "Premium" : "Standard";

    return {
      ...emp,
      tax,
      netSalary,
      status
    };
  });

  console.log(
    `Payroll Processed: Total Net Payout is $${totalNetPayout} for ${employees.length} employees.`
  );

  setTimeout(() => {
    console.log(processedEmployees);
  }, 2000);
}

calculatePayroll(employees, taxLogic);

#Explanation:
-------------

Callback Function:
       function taxLogic(salary)

Tax Rules:

| Salary  | Tax |
| ------- | --- |
| > 5000  | 20% |
| <= 5000 | 10% |


Tax Calculation:
       const tax = emp.salary * taxRate;

Net Salary:
       const netSalary = emp.salary - tax;

Status Logic:
       netSalary > 4000


| Net Salary | Status   |
| ---------- | -------- |
| > 4000     | Premium  |
| <= 4000    | Standard |

Ternary Operator:
       condition ? trueValue : falseValue
       Used for status checking.

setTimeout():
       2000ms = 2 seconds
       Simulates delayed payroll processing.
