## 🗓️ Day 2 Summary

On Day 2, we continued JavaScript with more advanced concepts and modern syntax.

### Topics Covered:
1. Template Literals
          Concept: Uses backticks (`) and ${} for cleaner string interpolation.

2. Arrow Functions
          Concept: A shorter syntax for writing concise functions.
           Example:
           let sum = (x, y) => x + y;
           console.log(sum(15, 20)); // 35

3. Array Operations 
          Concept: Array Operations are method used to add,remove, or modify elements in array(push,pop,shift,includes,etc)
          
           Example:
           let colors = ["red", "green", "blue"];
           colors.push("yellow");    // Add to end
           colors.shift();           // Remove from start
           colors.splice(1, 1);      // Remove 1 item at index 1
           colors.includes("red");   // true

4. Math & Type Casting
          Concept: Handling calculations and converting data between types.

          Example:
          Math.floor(Math.random() * 100) + 1; // 1–100
          parseInt("11.6");  // 11
          parseFloat("11.6"); //11.6
          Number("5");  

5. High order Array methods
           Concept: Methods like map(), filter(), and find() are higher-order array methods that use a callback function to process and return elements from an array.

            Example:
            let arr = [1, 2, 3, "Nandini"];
            let nums = arr.filter(x => typeof x === "number").map(y => y * 2);
            // nums: [2, 4, 6]

6. Reduce method
            Concept: Method that processes an array and reduces it to a single value using a callback function.

            Example:
            let fruits = ["apple", "banana", "apple","mango","pinneapple","mango","apple"];
            let count = fruits.reduce((acc, curr) => {
            acc[curr] = (acc[curr] || 0) + 1;
            return acc;
            }, {});
            console.log(count);

7. DOM Manipulation
              Concept: DOM manipulation is the process of using JavaScript to access and modify HTML elements, content, and styles of a webpage.

-Linking HTML,CSS and JavaScript Files.

-Navigation between pages(using the links).

📘 Day 04 Summary: Asynchronous JavaScript

## Objective

To understand how JavaScript handles asynchronous operations using:

Timers (setTimeout, setInterval)
Callback functions
Promises
Async/Await

* * * * * * ** * * * * * * * * * * * * * ** * * * ** * * * * ** * * ** *  *

1. setTimeout:
        setTimeout() is used to execute a function once after a specific delay.

- Example:
        setTimeout(() => {
        console.log("Runs after 2 sec");
        }, 2000);

- Key Points:
Delay is in milliseconds
Executes only once
Non-blocking (runs in background)

* ** * * * * * * ** * * * ** * * ** * * * * * * ** ** * ** * ** * * ** ** * * * 

2. setInterval:
        setInterval() runs a function repeatedly after a fixed interval.

- Example:
        let id = setInterval(() => {
        console.log("Runs every 1 sec");
        }, 1000);

- stop interval:
        clearInterval(id);


clearInterval() stops execution
If used immediately, interval may not run
 
- Better Usage:
        setTimeout(() => {
        clearInterval(id);
        }, 5000);

* * * * * *  * * * * * ** * * * * * ** ** * * ** * * ** * * * ** * * * * * * * ** 

3. Callback Function:
      A callback is a function passed as an argument to another function and executed later.

- Example:
     function getData(callback) {
     setTimeout(() => {
     console.log("Data fetched");
     callback();
     }, 1000);
     }

     getData(() => console.log("Next step"));

 Output:
     Data fetched
     Next step

- Limitation:
     Can lead to callback hell (nested callbacks)

* * ** * * * ** * ** * * ** * * * * ** * ** * * * * * * ** * * ** * * * * * * * * *

4. Promises:
     A Promise represents the result of an asynchronous operation.

- States:
     Pending
     Resolved
     Rejected

- Example:
     let myPromise = new Promise((resolve, reject) => {
     let success = true;

     if (success) resolve("Success");
     else reject("Error");
     });

- Handling:
myPromise
    .then(result => console.log(result))
    .catch(error => console.log(error));

* * * * * * * * * * * * * * * * * * * * * ** ** * * ** * * *** * * * * * * * *  ** * 

5. Async/Await:
     A modern way to handle promises, making code clean and readable.

- Example:
     async function run() {
    try {
        let result = await myPromise;
        console.log(result);
    } catch (err) {
        console.log(err);
    }
}

      run();

- Key Learnings:
     JavaScript handles async tasks without blocking execution
     Timers help schedule tasks
     Callbacks are basic async handling
     Promises improve structure and readability
     Async/Await makes code simple and professional
